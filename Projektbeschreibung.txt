Projekt: Direkte Kommmunikation mit dem Roboter

- Was wir machen:
  Wir schauen uns an, wie wir Roboter mit unserer Stimme kontrollieren können. Um in das Spielgeschehen einzugereifen.

- Wo wir stehen:
  In der Codebase gibt es keine Grundlegenden Code Stücke die man verwenden könnte.
  Es wird eine voice recognition als open source Code benötigt.

- Was wir vorhaben:
  - Technische Sachen:
    Es wird eine leichte run time benötigt.
    Der Roboter soll nicht viel verstehen nur Grundlegende Wörter, wie "Frohe Weihnachten".
    Ein signal voice filter wird benötigt sodass er nicht alles wahrnimmt sondern nur die Befehle / tiefe töne 
    Man braucht nur eine geringe abtastrate / 12 kHz / 30 fps 
    threads angeben 
  
  - Wie wir das machen:
    Wir schauen uns an, was wir schon an Code haben und wo wir neuen Code herbekommen:
    In git hub/Web nach open source programmen suchen (voice recognition).
    https://github.com/AI-TOOLKIT/VoiceBridge
    https://fosspost.org/open-source-speech-recognition/
    https://www.knowledgenile.com/blogs/what-are-some-of-the-best-open-source-speech-recognition-software
  	
  - Erstes Ziel:
    Einen Roboter dazu bringen, das er auf Frohe Weihnachten reagiert.
    Sodass er durch 74-feature-beepcomms eine signal reaktrion ausgibt.

- Was als Nächstes passiert:
  - Grober Plan:
    Zusammen suchen von Code Stücken die zur voice recognition beitragen.
    Da der Code nicht mit WSL getestet werden kann muss ein dual boot durchgeführt werden.
    Sodass der Code auf Linux getestet werden kann.
